# Venue Shortlist (Systems/Engineering First)

## Tier A: Practical Systems and Engineering Fit
1. IEEE Software
- Fit: architecture + implementation + lessons-learned framing.
- Strength required: clear practitioner value and reproducibility package.

2. ACM Queue (or similar practitioner venue)
- Fit: real-world systems narrative with operational constraints.
- Strength required: concrete engineering trade-offs and production lessons.

3. USENIX ;login: / practitioner tracks (as applicable)
- Fit: operational systems perspective with workflow integrity emphasis.
- Strength required: strong deployment and reproducibility discussion.

## Tier B: Security/Governance-Oriented Academic Fit
1. ACM SACMAT (Access Control Models and Technologies)
- Fit: if emphasis is on mixed RBAC model and policy enforcement/state transitions.
- Gap to close: formalize authorization guarantees and failure cases.

2. IFIP SEC workshop tracks
- Fit: governance, auditability, and compliance system design.
- Gap to close: stronger threat model and controls analysis.

## Tier C: Future Extension Venues (After Added Novelty)
1. SIGIR/ECIR-style tracks (if relevance ranking/IR model is added).
2. CSCW/CHI-style tracks (if controlled user study and cognitive metrics are added).

## Recommendation
- First submission target: IEEE Software style systems/practice paper.
- Backup target: governance/security workshop with stronger policy-enforcement framing.
- Escalation path: add ML or HCI novelty after baseline systems publication.
